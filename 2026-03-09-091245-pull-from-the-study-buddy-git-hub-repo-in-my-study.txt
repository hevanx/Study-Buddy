> is the database going to be different on this machine versus my other     
  one? like the actual content in it                                        

● Yes — the database data lives inside the Docker container's volume, which is
  local to each machine. So:

  - Schema (tables, columns) — same on both, since it's defined in your repo
  - Data (users, progress, scores) — completely separate per machine. Whatever
  you register/create on one machine won't appear on the other

  If you want them to stay in sync, you'd need to either:
  1. Export the DB on one machine (mysqldump) and import it on the other
  2. Use a hosted/cloud database instead of a local one

  For a dev setup this is usually fine — just expect to re-register accounts on
  each machine.
